HTTP/HTTPS

HTTP (kort för Hypertext Transfer Protocol) är ett protokoll som används för att överföra data från hemsidor på WWW (World Wide Web) i internet. Syftet med HTTP var att överföra HTML-sidor från webbservrar till klienter. HTTP bygger på en förfrågan/svar mellan klienten och servern. En klient som vill hämta en HTML-fil (gå in i en hemsida), en bild eller en fil skickar en HTTP-förfrågan till servern bestående av instruktioner i formen av en sträng. Strängen innehåller information som vilken version av HTML som används och vad klienten vill hämta, till exempel en bild eller HTML-fil. När servern får tag på förfrågan så skickar den en HTTP-svar till klienten. Svaren innehåller information som HTTP responsstatus (en kod som hjälper till att identifiera problem om det skedde några) och data som klienten förfrågade.

HTTP har 8 olika kommandon (instruktioner) som klienten kan skicka till en HTTP-server. Varje kommando ger olika svar:

  • GET - Ber servern att skicka den begärda filen till klienten.
  • HEAD - Ber servern att skicka information om den begärda filen utan själva innehållet.
  • POST - Sänder data till servern, vanligtvis från en API eller ett formulär.
  • PUT - Skapar eller uppdaterar en resurs på servern med data som skickas.
  • DELETE - Begär servern att ta bort en resurs.
  • PATCH - Används för att göra delvisa uppdateringar av en resurs på servern.
  • OPTIONS - Begär information om vilka metoder får användas på en resurs.
  • CONNECT - Skapar en tunnel till servern, ofta för att hantera HTTP-trafik.

HTTP är föråldrat och borde inte användas på grund av säkerhetsproblem. Den har blivit ersatt med HTTPS (Hypertext Transfer Protocol Secure) vilket krypterar data som transporteras mellan servern och klienten.
